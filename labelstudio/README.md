## 更改日志与发布版本
发布版本基于语义化版本控制，并使用 ``MAJOR.MINOR.PATCH`` 的格式。简而言之，版本将根据以下规则进行递增：

- ``MAJOR``：不兼容或重大变更。
- ``MINOR``：向后兼容的新功能和增强。
- ``PATCH``：向后兼容的bug修复和包更新。
